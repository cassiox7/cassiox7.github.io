1.1. Programa - Regiões

Utilizando o programa exemplos/pixels.cpp como referência, implemente um programa regions.cpp. Esse programa deverá solicitar ao usuário as coordenadas de dois pontos \(P_{1}\) e \(P_{1}\) localizados dentro dos limites do tamanho da imagem e exibir que lhe for fornecida. 1.5.1. Código

trocaregioes.cpp
#include <iostream>
#include <opencv2/opencv.hpp>

using namespace cv;
using namespace std;

int main(int, char** argv){
  Mat image;
  int auxiliar;
  int largura, altura;

  image= imread(argv[1],CV_LOAD_IMAGE_GRAYSCALE);
  if(!image.data)
    cout << "A imagem não pode ser carregada" << endl;

  largura =  image.size().width;
  altura  =  image.size().height;

  namedWindow("janela",WINDOW_AUTOSIZE);

  imshow("janela", image);
  waitKey();

  // Trocando os quadrantes
  for(int i=0;i<altura;i++)
  {
    for(int j=0;j<(largura/2);j++)
    {
        // Trocando os quadrantes 1 e 4
        auxiliar = image.at<uchar>(i,j); //Guardano o pixel atual para fazer a troca
        if (i < altura/2)
        {
          image.at<uchar>(i,j) = image.at<uchar>(i + altura/2 , j + largura/2);
          image.at<uchar>(i + altura/2 , j + largura/2) = auxiliar;
        }

        // Trocando os quadrantes 2 e 3
        else
        {
          image.at<uchar>(i,j) = image.at<uchar>(i - altura/2 , j + largura/2);
          image.at<uchar>(i - altura/2 , j + largura/2) = auxiliar;
        }

    }
  }

  imshow("janela", image);
  imwrite("Troca Regiões.png", image);
  waitKey();
  return 0;
}

Com isso pode-se fazer a troca pixel a pixel por meio da variável auxiliar com os pixels do 4º quadrante.

O pixel atual do 1º quadrante (i,j) recebe o valor do pixel do 4º quadrante, onde o mesmo pode ser achado por meio da observação, pois o 1º pixel do 1º quadrante será trocado com o 1º pixel do 4º quadrante, que é o mesmo que somar o valor da atual altura i com a metade da mesma e soma da atual largura j com a metade da mesma.

  5. O pixel atual do 4º quadrante recebe o valor do pixel do 1º quadrante que estava armazenado na variável auxiliar.

  6. Caso já se tenha passado da metade da altura da imagem deve-se começar a fazer a troca do 3º quadrante com o 2º quadrante, pois como o i estará com o valor acima do valor total do tamanho da imagem e como a largura da imagem o j só vai até a metade, essa parte do código está restrita ao 3º quadrante . Onde o pixel atual do 3º quadrante (i,j) recebe o valor do pixel do 2º quadrante, onde o mesmo pode ser achado por meio da observação, pois o 1º pixel do 2º quadrante será trocado com o 1º pixel do 3º quadrante, que é o mesmo que subtrair o valor da atual altura i com a metade da mesma e soma da atual largura j com a metade da mesma.

  7. O pixel atual do 2º quadrante recebe o valor do pixel do 3º quadrante que estava armazenado na variável auxiliar.

Utilizando o programa exemplos/histogram.cpp como referência, implemente um programa motiondetector.cpp. Este deverá continuamente calcular o histograma da imagem (apenas uma componente de cor é suficiente) e compará-lo com o último histograma calculado. Quando a diferença entre estes ultrapassar um limiar pré-estabelecido, ative um alarme. Utilize uma função de comparação que julgar conveniente.
